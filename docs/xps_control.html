class place.plugins.xps_control.xps_control.Stage(config)¶

Bases: place.plugins.instrument.Instrument

The base class for all movement stages.

This class provides access to the XPS controller that controls the movement of stages. Movement that is specific to a subset of stages should be written into the subclasses.

cleanup(abort=False)¶

Stop stage movement and end scan.

For us, this simply means closing the connection to the XPS controller.

Parameters:abort (bool) – indicates the scan has been stopped rather than having finished normally
config(metadata, updates, directory)¶

Configure the stage for a scan.

For a movement stage, configuring means setting up all the internal values. It does not mean that we move to the first position. No actual movement should happen until the update() method is called.

At this time, all we need to do is initialize all our class variables and connect to the XPS controller.

Parameters:
  • metadata (dict) – metadata for the scan
  • updates (int) – the number of update steps that will be in this scan
  • directory (str) – the location to save any data for this instrument
update(metadata, update_number, socket=None)¶

Move the stage.

The class uses an iterator to keep track of the position of the stage. So, all we need to do is call next() on it, and it will return the next position.

We will then move the stage and ask the controller to return to us the actual position the stage settled at. We will save this position into the header.

Parameters:
  • metadata (dict) – metadata for the scan
  • update_number (int) – the current update count
  • socket (websocket) – connection to the webapp plot frame
